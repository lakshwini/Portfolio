Ce que j'ai mis en place

Deux services, trois environnements

L'objectif était de supprimer toutes les opérations manuelles entre l'écriture du code et le déploiement en production.

J'ai développé et containerisé deux services indépendants : Spring Boot + MongoDB pour l'upload d'images, et Flask + MySQL pour la gestion de profils utilisateurs.

Le point le plus délicat : les bases de données. Local et test partagent les mêmes données ; la production reste isolée.

CI — Intégration Continue
À chaque push sur main : build, tests, images Docker et push Docker Hub. Entièrement automatisé.
CD — Déploiement Continu
Déploiement via deploy.sh qui pull les images et relance les containers.
