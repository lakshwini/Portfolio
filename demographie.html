Le projet

Deux livrables, une même donnée

J'ai travaillé sur les données du recensement INSEE 2019 pour analyser la structure démographique de la France. L'idée : un outil où sélectionner une région met à jour tout instantanément.

J'ai produit deux livrables indépendants. L'application web est codée en JavaScript — Plotly.js pour les graphiques, PapaParse pour les CSV, SheetJS pour lire les Excel dans le navigateur.

Le dashboard Power BI repose sur un modèle en étoile. J'ai écrit des mesures DAX pour calculer la densité et afficher la pyramide des âges.

struct_pop…âge_2019.xlsx
Population par région, département, sexe et tranche d'âge.
struct_pop…csp_2019.xlsx
Répartition des catégories socio-professionnelles.
superficies_france.xlsx
Superficie en km² par région pour calculer la densité.
codes_noms_régions.csv
Référentiels géographiques INSEE.
Deux approches

App web vs Power BI

Application Web JS
  • ✦ 785 lignes de JavaScript, zéro framework
  • ✦ Chargement asynchrone des 5 fichiers sources
  • ✦ 8 graphiques Plotly.js avec filtres en temps réel
  • ✦ Filtre région → département en cascade
  • ✦ Fonctionne en local sans serveur
Dashboard Power BI
  • ✦ Modèle en étoile — 2 tables de faits, 2 dimensions
  • ✦ 4 mesures DAX : Population, Densité, Pop H, Pop F
  • ✦ Pyramide : population masculine en négatif via DAX
  • ✦ 3 pages : Dashboard national, Âge/Sexe, CSP
  • ✦ Carte de France cliquable par département
Les formules Power BI

Mesures DAX utilisées

Densité (hab/km²)
Densité =
VAR superficie = CALCULATE(SUM(Superficies[superficie_km2]))
RETURN IF(superficie > 0, DIVIDE([Population], superficie), BLANK())
Pop Hommes (négatif pour pyramide)
Pop Hommes =
VAR total = CALCULATE(SUM(Population[Pop_Hommes_2019]))
RETURN IF(total > 0, -total, BLANK())
